## Running TF Locally ##

1. Install gcloud cli
2. Run command, gcloud auth application-default login, to create credentials in ~/.config/gcloud/application_default_credentials.json (This will be picked up by Terraform)

## Running using GH Actions ##

# Setup Workload Identity Federation - for Github Actions and GCP Auth #

1. IAM & Admin > Workload Identity Federation
2. Create Pool
    name: github-pool
    ID: github-pool
    Location: global
3. Add Provider - OIDC
    Name: github-provider
    Issuer URI: https://token.actions.githubusercontent.com
    Allowed Audiences: Default
    Attribute Mapping:
        google.subject: assertion.sub
        attribute.repository: assertion.repository
    Attribute Conditions:
        attribute.repository.startsWith("kgacandole/")
4. Create a service account with below permissions:

5. Bind the service account to the identiy pool using below command:

# Allow GitHub OIDC identities to act as terraform-gcp-sa
gcloud iam service-accounts add-iam-policy-binding terraform-gcp-sa@project-235e2136-7c2e-4409-bad.iam.gserviceaccount.com \
  --role="roles/iam.workloadIdentityUser" \
  --member="principalSet://iam.googleapis.com/projects/390209572226/locations/global/workloadIdentityPools/github-pool/attribute.repository/kgacandole/github_arc"

# Allow Terraform to request access tokens (needed by provider)
gcloud iam service-accounts add-iam-policy-binding terraform-gcp-sa@project-235e2136-7c2e-4409-bad.iam.gserviceaccount.com \
  --role="roles/iam.serviceAccountTokenCreator" \
  --member="principalSet://iam.googleapis.com/projects/390209572226/locations/global/workloadIdentityPools/github-pool/attribute.repository/kgacandole/github_arc"

6. In the GCP bucket holding the state file, ensure that 'terraform-gcp-sa' SA has access to the bucket (Edit the IAM Policy condition).

7. Add binding on develoepr/terraform service account (otherwise, Kube Cluster creation will fail)

gcloud iam service-accounts add-iam-policy-binding 390209572226-compute@developer.gserviceaccount.com \
  --member="serviceAccount:terraform-gcp-sa@project-235e2136-7c2e-4409-bad.iam.gserviceaccount.com" \
  --role="roles/iam.serviceAccountUser"

# Project-level binding #

gcloud projects add-iam-policy-binding "project-235e2136-7c2e-4409-bad" \
  --member="serviceAccount:terraform-gcp-sa@project-235e2136-7c2e-4409-bad.iam.gserviceaccount.com" \
  --role="roles/container.admin" \
  --condition="expression=true,title=always_allow,description=No restrictions"

8. Add below binding for node pool creation

gcloud iam service-accounts add-iam-policy-binding terraform-gcp-sa@project-235e2136-7c2e-4409-bad.iam.gserviceaccount.com \
  --member="serviceAccount:terraform-gcp-sa@project-235e2136-7c2e-4409-bad.iam.gserviceaccount.com" \
  --role="roles/iam.serviceAccountUser"

gcloud projects add-iam-policy-binding "project-235e2136-7c2e-4409-bad" \
  --member="serviceAccount:terraform-gcp-sa@project-235e2136-7c2e-4409-bad.iam.gserviceaccount.com" \
  --role="roles/compute.viewer"
